############################################################
# Executing the gen-docs.sh script
############################################################
The "gen-docs.sh" script is the script for generating documentation. Currently it is just a wrapper around Sphinx commands

Because the script auto-determines several things (project name, documentation directory, etc) you should only execute it from a command line from inside this directory like so:
./gen-docs.sh

If new Python code files are added to the project (and need documentation) a Sphinx <morule name>.rst template file will need to be created (by hand) in ./source and the "./gen-docs.sh" command re-run to render it.

Note: The script currently accepts a single argument "r", for regenerating *all* documentation from scratch:
./gen-docs.sh r
This should *only be done onne time, once a majority of the project code files are written* to generate the documentation framework  and basic *.rst files for every python file/module in the project directory. Once this is generated there should be no need to run this again.

# Sphinx configuration
############################################################

##########
# Directories of interest
##########
./source
  All Sphinx templates (*.rst files) that are used to build each page.  There shodul be one *.rst file for each pyton module, and each gets rendered into a documentation page.  There are additional *.rst files to handle generating of index and other pages.

./source/_build/html
  This is the root directory of where autogenerated html files are put.

##########
# Files of interest
##########
./source/Makefile
  (Autogenerated by "./gen-docs.sh r"). Contains all the additional options when executing make.  See ./gen-docs.sh for what options it is using for make.

./source/index.rst
  (Autogenerated by "./gen-docs.sh r"). This is the main Sphinx template for the "entry point" of the documentation.

./source/conf.py
  (Autogenerated by "./gen-docs.sh r"). This is *the* configuration file that globally directs the Sphinx documentation process.

./source/_build/html/index.html
  Entry point HTML file for viewing documentation

############################################################
# Usage in github for gh-pages
############################################################
- ***One time only*** use Github pages to create a gh-pages "tracking branch" (it also activates a website with some default files)
- Copy the ./source/_build/html directory to a directory *outside* of the github project.
- Change project branch to gh-pages
  Note: NEVER merge the gh-pages branch with the main branch!
- Copy the html directory to the root of the gh-pages branch
- ***One time only***
  1. Edit the "index.html" file in the branch. Add a link to the ./html/index.html page
  2. IMPORTANT! Create a new, empty file called ".nojekyll" at the the root folder of the "gh-pages" branch.  This is because github parses all docs with jekyll and we are using Sphinx. NOTE: it may be "hidden"
  3. Update the gh-pages/index.html file to add an HTML link as "API documenation" to point to the Sphinx api-docs index.html:
   The full URL is: http:/<ProjectUserName>.github.io/<ProjectName>/html/index.html

- Check in the gh-pages branch

############################################################
# "Standard" changes to default Sphinx configuration
############################################################
1. In ./source/conf.py: Add project module path
   Note: Look for commented out sys.path.insert section in conf.py:
   Add:
   sys.path.insert(0, os.path.abspath("../../"))

2. In ./source/conf.py: Change the html_theme to "classic". See: http://sphinx-doc.org/theming.html#builtin-themes
   Change:
   html_theme = 'classic'

3. In ./source/conf.py: Update "Exclude patterns" list for project directories + files you *do not* want to document.  Note: These still may be parsed/loaded if they are dependencies for the project code.
   Change:
exclude_patterns = [
  '_build',
  '*.libs.*',
  '*.resources.*',
  '*.docs.*',
  '*.kivy.*',
  '*.user.*',
]
